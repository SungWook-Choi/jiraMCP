# 작업 지시서: v0.4.0 1번 설정 CLI 추가

## 목적

이 문서는 사용자 설정 파일을 직접 편집하지 않고, CLI 질문 흐름으로 생성/수정할 수 있도록 `qwen-jira-config` 명령을 추가하는 작업 범위를 고정한다.

## 배경

현재 사용자는 아래 명령을 사용한다.

- `qwen-jira-server`
- `qwen-jira`

추가 요구사항은 사용자별 설정을 별도 명령으로 관리하는 것이다.

즉 사용자는 설정 항목을 파일 편집이 아니라 질문형 CLI로 변경할 수 있어야 한다.

## 요구사항

1. 새 명령어 이름은 `qwen-jira-config`로 한다.
2. 명령 실행 시 설정 파일이 없으면 새로 생성할 수 있어야 한다.
3. 명령 실행 시 기존 설정이 있으면 현재 값을 기준으로 수정 여부를 묻는다.
4. 질문 흐름은 항목별 `Y / N` 방식으로 진행한다.
5. `Y`를 선택하면 새 값을 입력받고 저장 후보에 반영한다.
6. `N`을 선택하면 기존 값을 유지하고 다음 질문으로 이동한다.
7. 마지막에는 저장 여부를 한 번 더 확인한다.

## 질문 대상 항목

이번 단계에서 묻는 항목은 아래 3개로 제한한다.

1. `PORT`
2. `assignee all` 포함 대상 담당자 목록
3. 결과 출력 폴더 경로

## 설정 파일 기준

이번 작업에서 설정 파일 위치와 키 이름은 아래로 확정한다.

1. 설정 파일 위치: `<사용자 홈>/.qwen-jira-mcp/config.json`
2. 설정 파일은 `UTF-8` JSON 파일로 저장한다.
3. 설정 파일의 부모 디렉터리가 없으면 자동 생성한다.
4. 저장 키는 아래 이름만 사용한다.

```json
{
  "serverPort": 3000,
  "assigneeAllInclude": [
    "Alice Kim",
    "Bob Lee"
  ],
  "resultOutputDir": "./output"
}
```

`qwen-jira-config`는 이번 범위에서 위 세 키 외의 값을 묻거나 저장하지 않는다.

## 예시 UX

```text
Port를 수정하시겠습니까? (Y/N)
Y
Port를 입력하세요:
3101

전체 조회 포함 담당자를 수정하시겠습니까? (Y/N)
Y
쉼표로 구분해서 입력하세요:
Alice Kim, Bob Lee

결과 출력 폴더를 수정하시겠습니까? (Y/N)
N

저장하시겠습니까? (Y/N)
Y
```

## 범위

1. `qwen-jira-config` 진입점 추가
2. 질문형 입력 흐름 구현
3. 설정 파일 생성/수정/저장
4. 잘못된 입력에 대한 기본 검증
5. `package.json`의 `bin` 항목에 `qwen-jira-config` 추가
6. 설정 파일 경로와 명령 사용법 README 반영

## 제외 범위

1. Jira 접속 정보 입력
2. Web UI
3. 고급 검증 UI
4. 암호화 저장

## 예상 변경 대상

1. `bin/`
2. `src/cli/` 또는 별도 `src/config-cli/`
3. 설정 파일 입출력 유틸
4. `README.md`
5. 필요 시 `.env.example`

## 완료 기준

1. `qwen-jira-config` 명령이 실행된다.
2. 설정 파일이 없을 때 생성된다.
3. 설정 파일이 있을 때 수정 흐름이 동작한다.
4. 세 항목에 대해 `Y / N` 질문과 입력이 가능하다.
5. 최종 저장 확인 단계가 있다.
6. 저장 결과가 `<사용자 홈>/.qwen-jira-mcp/config.json`에 `UTF-8` JSON으로 기록된다.
7. `npm run build`와 `npm run typecheck`가 통과한다.

## 검수 기준

1. 질문 흐름이 사용자 친화적이고 단순하다.
2. 설정 파일 직접 편집 없이 기본 사용이 가능하다.
3. 이번 범위 밖 설정 항목은 노출하지 않는다.
