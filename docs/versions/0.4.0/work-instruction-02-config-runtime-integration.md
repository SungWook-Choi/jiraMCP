# 작업 지시서: v0.4.0 2번 설정 반영 로직 추가

## 목적

이 문서는 사용자 설정 파일의 값을 실제 서버 실행, 전체 담당자 조회, 결과 출력 경로에 반영하는 작업 범위를 고정한다.

## 배경

설정 CLI만 추가하면 사용자는 값을 저장할 수 있지만, 실제 동작에서 사용되지 않으면 기능 가치가 없다.

따라서 이번 작업은 설정 파일에 저장된 세 항목을 실행 로직에 연결하는 것을 목표로 한다.

## 반영 대상

1. `PORT`
2. `assignee all` 포함 대상 담당자 목록
3. 결과 출력 폴더 경로

## 설정 파일 기준

이 작업은 `work-instruction-01-config-command.md`에서 확정한 사용자 설정 파일을 그대로 사용한다.

1. 설정 파일 위치: `<사용자 홈>/.qwen-jira-mcp/config.json`
2. 설정 파일은 `UTF-8` JSON 파일로 읽는다.
3. 사용하는 키는 아래 3개로 제한한다.

```json
{
  "serverPort": 3000,
  "assigneeAllInclude": [
    "Alice Kim",
    "Bob Lee"
  ],
  "resultOutputDir": "./output"
}
```

4. Jira 인증 정보는 이 설정 파일에서 읽지 않는다.
5. 설정 파일이 없으면 기본값으로 동작하되, `assigneeMode=all`은 포함 대상 목록이 없다는 안내를 반환해야 한다.

## 요구사항

### 1. PORT

1. `qwen-jira-server` 실행 시 사용자 설정 파일에 `serverPort`가 있으면 그 값을 기본 포트로 사용한다.
2. 환경변수 `PORT`가 있으면 사용자 설정보다 우선한다.
3. 설정 파일에도 환경변수에도 값이 없으면 기본값 `3000`을 사용한다.
4. `qwen-jira` CLI가 로컬 서버 주소를 자동 결정할 때도 사용자 설정 파일의 `serverPort`를 반영한다.
5. CLI 서버 주소 우선순위는 환경변수 `QWEN_JIRA_API_BASE_URL`, 환경변수 `LOCAL_SERVER_API_BASE_URL`, 사용자 설정 파일의 `serverPort`, 기본값 `http://127.0.0.1:3000` 순서로 한다.

우선순위:

1. 환경변수 `PORT`
2. 사용자 설정 파일
3. 기본값 `3000`

CLI 서버 주소 우선순위:

1. 환경변수 `QWEN_JIRA_API_BASE_URL`
2. 환경변수 `LOCAL_SERVER_API_BASE_URL`
3. 사용자 설정 파일의 `serverPort`로 만든 `http://127.0.0.1:<serverPort>`
4. 기본값 `http://127.0.0.1:3000`

### 2. 전체 담당자 조회 포함 대상

1. `assigneeMode=all` 조회 시 설정 파일의 `assigneeAllInclude` 목록을 사용한다.
2. 포함 대상 목록이 있으면 해당 담당자들만 조회 대상에 포함한다.
3. 포함 대상 목록이 비어 있으면 현재 동작을 그대로 fallback 하지 않고, 사용자에게 설정이 필요하다는 안내 오류를 반환한다.

이 항목은 “all”의 의미를 무제한 전체가 아니라, 사용자 설정으로 정의된 전체로 재정의하는 작업이다.

### 3. 결과 출력 폴더

1. Markdown 등 파일 출력 시 설정 파일의 `resultOutputDir` 값을 사용한다.
2. 설정값이 없으면 현재 기본 출력 경로를 사용한다.
3. 대상 폴더가 없으면 자동 생성할 수 있어야 한다.

## 완료 기준

1. 사용자 설정 파일의 포트가 서버 실행에 반영된다.
2. 사용자 설정 파일의 포트가 CLI의 기본 로컬 서버 주소에도 반영된다.
3. `assignee all` 조회가 설정된 담당자 목록 기준으로 동작한다.
4. 결과 출력 폴더가 설정값 기준으로 반영된다.
5. 환경변수와 사용자 설정의 우선순위가 문서와 코드에서 일치한다.
6. `npm run build`와 `npm run typecheck`가 통과한다.

## 검수 기준

1. 설정 파일이 실제 동작에 반영된다.
2. `PORT` 우선순위 규칙이 명확하다.
3. `assignee all`의 의미가 더 정확해진다.
4. 결과 출력 경로가 사용자 설정으로 제어된다.
