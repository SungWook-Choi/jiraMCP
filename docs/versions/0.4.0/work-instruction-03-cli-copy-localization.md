# 작업 지시서: v0.4.0 3번 CLI 문구 통일

## 목적

현재 `qwen-jira` CLI 사용자 흐름에는 영어와 한글 문구가 섞여 있다.
이번 작업은 사용자가 보는 CLI 표시 문구를 한국어 중심으로 통일해 사용 흐름을 자연스럽게 만드는 것을 목표로 한다.

## 배경

0.4.0에서 설정 CLI와 런타임 설정 반영이 추가되면서 실제 사용자가 보는 터미널 문구가 늘어났다.
현재 일부 문구는 `Select query mode`, `Assignee`, `Save result as Markdown?`, `Please choose...`처럼 영어이고, 일부 문구는 `첫 작업을 선택하세요`, `프로젝트 이름으로 검색`처럼 한국어다.

한 화면 안에서 두 언어가 섞이면 사용자가 같은 개념을 다른 것으로 오해할 수 있으므로, 표시 문구를 한 방향으로 정리한다.

## 범위

1. `qwen-jira` CLI의 사용자 표시 문구를 한국어 중심으로 통일한다.
2. `qwen-jira-config` 설정 CLI의 사용자 표시 문구도 같은 톤으로 점검한다.
3. CLI 오류 메시지 중 사용자가 직접 보는 문구를 한국어로 정리한다.
4. README의 사용 설명에서 CLI 선택지 표현을 한국어 표시 문구와 맞춘다.

## 통일 기준

1. 사용자가 보는 문장과 질문은 한국어로 작성한다.
2. 내부 enum 값, API 필드명, 코드 식별자는 변경하지 않는다.
3. 필요한 경우 기술 값은 괄호 안에 함께 표시할 수 있다.
   - 예: `담당자 기준 조회 (assignee)`
   - 예: `이번 주 (this_week)`
4. `Y/N`, URL, 파일 경로, 환경변수명, 명령어명은 원문을 유지한다.
5. Markdown 결과 본문은 이번 작업 범위에서 필수 변경 대상이 아니다. 단, CLI 저장 성공/실패 메시지는 한국어로 통일한다.

## 제외 범위

1. Query Schema enum 이름 변경
2. API 요청/응답 필드명 변경
3. Jira API 호출 로직 변경
4. MCP 응답 문구 변경
5. README 전체 재작성

## 예상 변경 대상

1. `src/cli/cli.service.ts`
2. `src/cli/cli-api.client.ts`
3. `src/config-cli/config-cli.service.ts`
4. `README.md`

## 완료 기준

1. `qwen-jira` 대화형 흐름에서 주요 질문/안내/오류 문구가 한국어 중심으로 통일되어 있다.
2. `qwen-jira-config` 문구와 톤이 크게 어긋나지 않는다.
3. 내부 enum 값과 API 필드명은 변경되지 않는다.
4. `npm run typecheck`와 `npm run build`가 통과한다.

## 검수 기준

1. 한 화면 안에서 불필요하게 영어/한글이 섞이지 않는다.
2. 영어 기술 값이 필요한 경우 한국어 설명과 함께 표시된다.
3. 기존 CLI 선택 흐름과 동작은 유지된다.
4. 범위 밖 기능 변경이 없다.
