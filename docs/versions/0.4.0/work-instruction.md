# 작업 지시서: v0.4.0 사용자 설정 기능 추가

## 문서 목적

이 문서는 `qwen-jira-mcp@0.4.0` 범위에서 추가할 사용자 설정 기능의 기준 문서다.

현재 `0.3.0`까지는 Jira 접속 정보와 기본 동작이 환경변수 중심으로 관리된다.  
이번 단계에서는 민감정보는 계속 환경변수로 두고, 사용자 운영 편의 항목만 별도 설정 파일과 설정 CLI로 관리하는 방향을 확정한다.

## 버전 범위

이번 `0.4.0` 범위에서는 아래 두 작업 항목을 다룬다.

1. `qwen-jira-config` 설정 명령 추가
2. 사용자 설정 파일을 실제 서버/CLI/조회 로직에 반영

## 운영 원칙

이번 단계에서 아래 원칙을 고정한다.

1. Jira 접속 정보는 계속 환경변수로 관리한다.
2. 설정 파일에는 민감정보를 저장하지 않는다.
3. 사용자 설정은 `qwen-jira-config` 명령으로 생성/수정한다.
4. 사용자가 직접 JSON 파일을 편집하는 흐름은 기본 UX로 요구하지 않는다.

환경변수로 유지하는 항목 예시:

- `JIRA_BASE_URL`
- `JIRA_EMAIL`
- `JIRA_API_TOKEN`
- 필요 시 `JIRA_PROJECT_KEY`
- 필요 시 `JIRA_DEFAULT_PERIOD`

설정 파일로 관리할 항목:

1. `PORT`
2. `assignee all` 조회 시 포함 대상 담당자 목록
3. 결과 출력 폴더 경로

## 설정 파일 방향

설정 파일은 사용자별 운영 설정만 담는다.

권장 예시:

```json
{
  "serverPort": 3000,
  "assigneeAllInclude": [
    "Alice Kim",
    "Bob Lee"
  ],
  "resultOutputDir": "./output"
}
```

정확한 파일명과 위치는 세부 지시서에서 확정한다.

이번 문서에서는 아래 항목을 다루지 않는다.

2. Web UI 설정 화면
3. MCP 설정 연동
4. 암호화 저장
5. 팀/조직 단위 중앙 설정 관리
