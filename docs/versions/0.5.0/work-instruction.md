# 작업 지시서: v0.5.0 주간보고 스케줄러 추가

## 문서 목적

이 문서는 `qwen-jira-mcp@0.5.0` 범위에서 추가할 주간보고 자동 생성 스케줄러의 기준 문서다.

현재 주간 업무 조회는 사용자가 직접 실행해야 한다.  
이번 단계에서는 `qwen-jira-server`가 실행 중일 때 지정된 요일 기준으로 주간 업무 조회를 자동 수행하고, 결과를 Markdown 파일로 저장하는 방향을 확정한다.

## 버전 범위

이번 `0.5.0` 범위에서는 아래 한 가지 작업 항목을 다룬다.

1. 주간보고용 Jira 주간 업무 조회 스케줄러 추가

## 운영 원칙

이번 단계에서 아래 원칙을 고정한다.

1. 스케줄러는 `qwen-jira-server`가 실행 중일 때만 동작한다.
2. 실행 요일은 사용자 설정 파일과 `qwen-jira-config`를 통해 관리한다.
3. 스케줄러는 기존 Jira 조회 코어를 재사용하고, 별도 조회 엔진을 새로 만들지 않는다.
4. 주간 업무 판정은 `updated`가 아니라 `changelog` 기준을 사용한다.
5. 자동 조회 담당자는 현재 로그인에 사용한 Jira API Token 소유자 본인으로 고정한다.
6. 결과 파일은 사용자 설정의 `resultOutputDir` 하위 주간보고 전용 폴더에 Markdown 확장자로 저장한다.
7. 결과 본문은 주간보고용 목록만 남기고 설명, 요약 문단, 부가 통계는 넣지 않는다.

## 설정 방향

이번 단계에서 사용자 설정 파일은 기존 키에 더해 아래 항목을 추가로 관리한다.

```json
{
  "serverPort": 3000,
  "resultOutputDir": "./output",
  "weeklyReportWeekday": "monday",
  "weeklyReportHour": 9
}
```

`weeklyReportWeekday`와 `weeklyReportHour`의 정확한 허용값, 시간 해석 기준, 비활성화 기준은 세부 지시서에서 확정한다.

## 세부 작업지시서

### 1. 주간보고 스케줄러

세부 구현 지시서는 `docs/versions/0.5.0/work-instruction-01-weekly-report-scheduler.md`를 따른다.

## 제외 범위

이번 문서에서는 아래 항목을 다루지 않는다.

1. Slack, Email, 메신저 알림 전송
2. Web UI 스케줄 관리 화면
3. 다중 스케줄 작업 추가
4. 분산 환경용 락 또는 다중 인스턴스 조정
5. Jira 조회 기준을 사용자가 자유롭게 조합하는 고급 스케줄러

## 산출물

1. `0.5.0` 기준 작업지시서 1건
2. 스케줄 실행 기준 정의
3. 설정 항목 추가 기준 정의
4. 출력 형식과 완료 기준 정의

## 검수 기준

1. 스케줄러 실행 조건이 문서에 명확하게 적혀 있다.
2. `qwen-jira-config`와 연동할 설정 항목이 문서에 반영되어 있다.
3. `changelog` 기반 주간 업무 판정 기준이 문서에 반영되어 있다.
4. Markdown 출력 형식이 구현자가 오해하지 않도록 충분히 구체적이다.
