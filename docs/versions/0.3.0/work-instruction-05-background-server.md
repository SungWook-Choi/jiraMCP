# 작업 지시서: v0.3.0 5번 백그라운드 서버 실행

## 문서 목적

이 문서는 `qwen-jira-mcp@0.3.0` 추가 개발 항목 중 5번인 서버 백그라운드 실행 구조의 구현 범위를 고정하기 위한 작업 지시서다.

상위 기준 문서는 `docs/versions/0.3.0/work-instruction.md`다.

## 작업 배경

현재 사용 흐름은 아래와 같다.

1. 사용자가 `qwen-jira-server`를 실행한다.
2. 서버가 포그라운드에서 터미널을 점유한다.
3. 사용자는 다른 터미널을 열어 `qwen-jira`를 실행해야 한다.

이 방식은 사용자에게 터미널을 2개 열어야 하는 불편을 만든다.

이번 작업에서는 `qwen-jira-server`를 실행해도 터미널 입력 권한이 사용자에게 바로 돌아오도록 바꾸는 것을 목표로 한다.

## 목표

1. `qwen-jira-server` 실행 시 서버를 백그라운드에서 실행한다.
2. 실행 후 호출한 터미널은 즉시 사용자에게 반환한다.
3. 백그라운드 서버의 상태를 확인할 수 있어야 한다.
4. 백그라운드 서버를 종료할 수 있어야 한다.
5. 백그라운드 서버의 로그를 파일로 남겨 운영 확인이 가능해야 한다.

## 사용자 흐름 기준

이번 작업 기준 사용자 흐름은 아래로 고정한다.

1. 사용자는 `qwen-jira-server`를 실행한다.
2. 서버는 백그라운드에서 실행된다.
3. 사용자는 같은 터미널에서 이어서 `qwen-jira`를 실행할 수 있다.
4. `qwen-jira`는 현재처럼 실행 후 종료되며, 완료되면 터미널 입력 권한을 사용자에게 돌려준다.

## 세부 규칙

1. `qwen-jira-server`의 기본 동작은 포그라운드가 아니라 백그라운드 실행이다.
2. 사용자가 별도의 daemon 옵션을 추가로 입력하지 않아도 된다.
3. `qwen-jira-server` 실행 시 이미 서버가 살아 있으면 중복 실행하지 않는다.
4. 중복 실행 시 사용자에게 이미 실행 중이라는 사실을 짧고 명확하게 안내한다.
5. 백그라운드 서버의 실행 정보는 재시작과 종료 판단이 가능하도록 내부 파일로 관리한다.
6. 백그라운드 서버 로그는 파일로 기록한다.
7. 로그에는 최소한 서버 시작, 리슨 성공, 오류, 종료 정보가 포함되어야 한다.
8. 서버 종료 명령은 사용자 수준에서 직접 사용할 수 있어야 한다.
9. 서버 상태 확인 명령은 사용자 수준에서 직접 사용할 수 있어야 한다.

## 권장 명령 동작

이번 작업에서는 아래 방향을 기준으로 구현한다.

1. `qwen-jira-server`
   서버를 백그라운드에서 시작한다.
2. `qwen-jira-server status`
   서버 실행 여부를 확인한다.
3. `qwen-jira-server stop`
   백그라운드 서버를 종료한다.

명령 수는 늘어나지만 일반 사용자는 주로 `qwen-jira-server`, `qwen-jira`만 사용하고, `status`, `stop`은 운영 보조 명령으로 본다.

## 변경 대상 기준

예상 변경 대상은 아래 범위 안으로 제한한다.

1. `bin`
2. `src/server-entry.ts`
3. 서버 실행 관련 bootstrap 또는 보조 유틸
4. 필요 시 설정 또는 로그 경로 관련 파일
5. README 또는 버전 문서

명시 지시가 없으면 Jira 조회 코어, Query Schema, Summary 구조를 이번 작업에서 함께 바꾸지 않는다.

## 로그 기준

로그는 백그라운드 실행 운영 확인용으로 사용한다.

최소 기준은 아래와 같다.

1. 로그 파일 경로가 일관되어야 한다.
2. 서버 시작 시각이 남아야 한다.
3. 포트와 호스트 정보가 남아야 한다.
4. 치명적 오류가 남아야 한다.
5. 종료 시각 또는 종료 이벤트가 남아야 한다.

## 종료 기준

종료 기능은 운영 필수 기능으로 본다.

최소 기준은 아래와 같다.

1. 사용자가 명령으로 서버 종료를 요청할 수 있다.
2. 이미 종료된 서버면 상태를 정리하고 사용자에게 안내한다.
3. PID 정보가 남아 있는데 실제 프로세스가 없으면 stale 상태를 정리한다.

## 완료 기준

아래 조건을 만족하면 이번 작업을 완료로 본다.

1. `qwen-jira-server` 실행 후 터미널이 즉시 반환된다.
2. 같은 터미널에서 이어서 `qwen-jira`를 실행할 수 있다.
3. 서버가 이미 실행 중이면 중복 실행되지 않는다.
4. `status` 명령으로 서버 상태를 확인할 수 있다.
5. `stop` 명령으로 서버를 종료할 수 있다.
6. 백그라운드 실행 중 로그 파일이 남는다.
7. 일반 사용자에게 필요한 안내 문구가 과도하게 복잡하지 않다.
