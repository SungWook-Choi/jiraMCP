# 작업 지시서: v0.3.0 1번 `[주간 이슈]` 댓글 조회

## 문서 목적

이 문서는 `qwen-jira-mcp@0.3.0` 추가 개발 항목 중 1번인 `[주간 이슈]` 댓글 조회 기능의 구현 범위를 고정하기 위한 작업 지시서다.

상위 기준 문서는 `docs/versions/0.3.0/work-instruction.md`다.

## 작업 배경

기존 시스템은 Jira 이슈의 `본문`과 `댓글`에서 `[주의사항]` 태그를 찾아 추출한다.

하지만 현재 운영 기준은 `주의사항`을 더 이상 사용하지 않고, 주간 보고 시 필수 확인 사항을 Jira 이슈 `댓글`의 `[주간 이슈]` 태그로 남기는 방식으로 바뀌었다.

예시 댓글:

```text
[주간 이슈]
작업하는데 이런 문제가 있습니다
```

따라서 이번 작업은 기존 `[주의사항]` 추출 규칙을 유지하지 않고, 댓글 전용 `[주간 이슈]` 추출 규칙으로 교체하는 것을 목표로 한다.

## 목표

1. Jira 이슈 댓글에서 `[주간 이슈]` 태그가 포함된 댓글을 식별한다.
2. `[주간 이슈]` 이후 내용을 추출한다.
3. 추출 결과를 사용자 출력에서 `주간 이슈` 항목으로 제공한다.
4. 기존 `[주의사항]` 기반 추출과 출력은 제거한다.

## 구현 범위

이번 작업 범위는 아래로 제한한다.

1. Jira 댓글 텍스트에서 `[주간 이슈]` 태그 식별
2. `[주간 이슈]` 이후 내용 추출
3. 같은 이슈 내 여러 `[주간 이슈]` 댓글 수집
4. Jira 검색 결과 정규화 필드명 변경
5. 콘솔 출력과 Markdown 출력에서 `주간 이슈`로 표시
6. 관련 문구와 예시 문서 정리

## 세부 규칙

1. 공식 태그는 정확히 `[주간 이슈]`로 본다.
2. 추출 대상은 Jira 이슈 `댓글`만 본다.
3. Jira 이슈 `본문`은 `[주간 이슈]` 추출 대상에서 제외한다.
4. 댓글 본문에 `[주간 이슈]`가 포함되면 해당 댓글을 주간 이슈 댓글로 간주한다.
5. `[주간 이슈]` 이후의 텍스트를 추출한다.
6. 같은 이슈에 `[주간 이슈]` 댓글이 여러 개 있으면 모두 수집한다.
7. 태그만 있고 뒤에 내용이 비어 있으면 빈 항목은 출력하지 않는다.
8. 동일한 내용이 중복으로 추출되면 중복 제거 여부는 기존 중복 제거 방식과 일관되게 처리한다.

## 제외 범위

아래 항목은 이번 작업 범위에서 제외한다.

1. `[주간 이슈]` 이외의 새 태그 추가
2. Jira 댓글 작성 기능 추가
3. CLI 입력 흐름 변경
4. `담당자 전체` 조회 구현
5. 프로젝트 진척률 계산 구현
6. 서버 데몬 실행 구조 변경
7. MCP 실제 조회 기능 확장

## 변경 대상 기준

예상 변경 대상은 아래 범위 안으로 제한한다.

1. `src/jira`
2. `src/summary`
3. 필요 시 관련 타입 또는 응답 구조
4. 필요 시 README 또는 버전 문서

명시 지시가 없으면 범위 밖 모듈 추가나 대규모 리팩토링은 금지한다.

## 산출물

이번 작업에서 최소한 아래 결과가 있어야 한다.

1. `[주간 이슈]` 댓글 추출 코드
2. `주간 이슈` 출력 반영
3. 기존 `[주의사항]` 출력 제거
4. 변경 내용이 반영된 문서 또는 예시

## 완료 기준

아래 조건을 만족하면 이번 작업을 완료로 본다.

1. 댓글에 `[주간 이슈]`가 있으면 해당 내용이 추출된다.
2. 이슈 본문에 같은 문구가 있어도 추출 대상에 포함하지 않는다.
3. 같은 이슈에 `[주간 이슈]` 댓글이 여러 개 있으면 모두 결과에 반영된다.
4. 출력 명칭이 `주의사항`이 아니라 `주간 이슈`로 표시된다.
5. 기존 `[주의사항]` 태그는 더 이상 결과 추출에 사용되지 않는다.
6. 구현 결과가 현재 Jira 검색 흐름을 깨지 않는다.

## 검증 기준

최소한 아래 항목을 검증해야 한다.

1. `[주간 이슈]`가 포함된 댓글이 있는 이슈에서 추출 성공
2. `[주간 이슈]`가 없는 이슈에서 빈 결과 처리
3. 여러 댓글이 있을 때 누락 없이 수집되는지 확인
4. 콘솔 출력 문구 확인
5. Markdown 출력 문구 확인
