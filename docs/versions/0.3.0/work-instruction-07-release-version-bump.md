# 작업 지시서: v0.3.0 릴리즈 버전 반영

## 1. 목적

이 문서는 현재 작업 결과를 `qwen-jira-mcp@0.3.0` 릴리즈 버전으로 정리하기 위한 작업 범위를 고정한다.

이번 작업의 목적은 실제 배포가 아니라, 패키지 버전과 관련 메타데이터를 `0.3.0`으로 맞추고 커밋 준비 상태로 만드는 것이다.

## 2. 배경

현재 배포 기준 버전은 `0.2.1`이며, `0.3.0` 범위의 기능 작업과 npm 배포 준비 작업이 진행되었다.

따라서 이번 단계에서는 버전 문자열을 실제 릴리즈 기준에 맞게 올리고, 패키지 메타데이터가 일관된지 확인해야 한다.

## 3. 범위

1. `package.json` 버전을 `0.3.0`으로 변경
2. 필요 시 lockfile 내 패키지 버전 동기화
3. 버전 문자열이 남아 있는 관련 파일 점검 및 최소 범위 수정
4. `npm run build`와 `npm pack --dry-run` 기준으로 버전 반영 확인

## 4. 세부 요구사항

1. 패키지 버전은 `0.2.1`에서 `0.3.0`으로 변경한다.
2. lockfile이 현재 패키지 버전을 포함하면 함께 맞춘다.
3. 기능 문서의 과거 버전 서술을 불필요하게 광범위 수정하지 않는다.
4. 실제 기능 로직은 건드리지 않는다.
5. 이번 작업은 버전 반영과 릴리즈 메타데이터 정합성까지만 다룬다.

## 5. 제외 범위

1. 실제 `npm publish`
2. 기능 변경
3. 라이선스 정책 변경
4. 새 문서 구조 개편

## 6. 예상 변경 대상

1. `package.json`
2. `package-lock.json`
3. 필요 시 README 또는 릴리즈 관련 최소 범위 파일

## 7. 검수 기준

1. `package.json` 버전이 `0.3.0`이다.
2. `package-lock.json`이 있으면 버전이 일관된다.
3. `npm run build`가 통과한다.
4. `npm pack --dry-run` 결과 패키지 버전이 `0.3.0`으로 나온다.

1. 변경 파일 목록
2. 반영한 버전 위치
3. `npm run build` 결과
