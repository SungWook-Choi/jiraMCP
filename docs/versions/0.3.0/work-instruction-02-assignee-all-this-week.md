# 작업 지시서: v0.3.0 2번 담당자 전체 조회

## 문서 목적

이 문서는 `qwen-jira-mcp@0.3.0` 추가 개발 항목 중 2번인 담당자 전체 조회 기능의 구현 범위를 고정하기 위한 작업 지시서다.

상위 기준 문서는 `docs/versions/0.3.0/work-instruction.md`다.

## 작업 배경

현재 CLI에서 `담당자 기준 조회`를 선택하면 바로 담당자 이름을 입력받는다.

하지만 요구사항은 담당자 기준 조회를 더 단순하고 명확하게 나누는 것이다.

사용자는 먼저 아래 두 가지 중 하나를 선택해야 한다.

1. `개인`
2. `전체`

`개인`은 현재와 같은 방식으로 동작하고, `전체`는 담당자명을 입력받지 않고 이번 주 기준 전체 담당자 이슈를 조회해야 한다.

## 목표

1. `담당자 기준 조회` 선택 시 `전체 / 개인` 옵션을 먼저 보여준다.
2. `개인` 선택 시 현재와 동일한 흐름을 유지한다.
3. `전체` 선택 시 담당자 입력 없이 전체 담당자 이슈를 조회한다.
4. `전체` 선택 시 기간은 `이번 주(this_week)`로 고정한다.
5. `전체` 선택 시 상태 `해야 할 일`, `완료`는 조회 결과에서 제외한다.

## 구현 범위

이번 작업 범위는 아래로 제한한다.

1. CLI에서 `담당자 기준 조회` 하위 옵션 추가
2. `개인 / 전체` 선택 흐름 추가
3. `전체` 선택 시 기간 입력 단계 생략
4. `전체` 선택 시 `this_week` 고정 처리
5. `전체` 조회용 Query/API 입력 처리
6. `전체` 조회용 Jira 검색 조건 반영
7. 상태 `해야 할 일`, `완료` 제외 조건 반영
8. 관련 문구와 사용 예시 문서 정리

## 세부 규칙

1. 이 작업은 `담당자 기준 조회`에만 적용한다.
2. 사용자가 `담당자 기준 조회`를 선택하면 먼저 `개인` 또는 `전체`를 선택하게 한다.
3. `개인` 선택 시 기존 담당자 입력 흐름과 기간 선택 흐름을 유지한다.
4. `전체` 선택 시 담당자 이름 입력 단계는 없다.
5. `전체` 선택 시 기간은 항상 `this_week`로 처리한다.
6. `전체` 선택 시 기간 선택 화면은 보여주지 않는다.
7. `전체` 조회 결과에서는 상태 `해야 할 일`, `완료`를 제외한다.
8. 상태 제외는 Jira 실제 상태명 기준으로 동작해야 한다.
9. 최소한 `해야 할 일`, `To Do`, `완료`, `Done`은 제외 대상으로 처리한다.
10. `전체` 조회는 담당자명이 없는 이슈가 있더라도 조회 자체가 깨지지 않아야 한다.

## 의미 기준

이번 작업에서 `전체`는 아래 의미로 본다.

1. 특정 담당자를 지정하지 않는다.
2. `이번 주` 기준으로 조회 가능한 담당자 이슈를 전체 대상으로 본다.
3. 결과는 기존 Jira 검색 결과 형식을 유지하되, 필요 시 assignee가 다른 이슈들이 함께 반환될 수 있다.

이번 작업에서는 `전체`를 위해 별도의 고급 그룹핑 UI를 추가하지 않는다.

## 제외 범위

아래 항목은 이번 작업 범위에서 제외한다.

1. `담당자 전체`의 다중 기간 선택 지원
2. `전체` 조회에서 사용자 목록을 먼저 불러와 선택하는 고급 UI
3. 프로젝트 기준 조회 흐름 변경
4. CLI 첫 진입을 `Jira 조회 / 댓글 입력`으로 나누는 작업
5. `[주간 이슈]` 댓글 조회 작업
6. 프로젝트 진척률 계산 작업
7. MCP 실제 조회 기능 확장

## 변경 대상 기준

예상 변경 대상은 아래 범위 안으로 제한한다.

1. `src/cli`
2. `src/api`
3. `src/query`
4. `src/jira`
5. 필요 시 README 또는 버전 문서

명시 지시가 없으면 대규모 리팩토링이나 새 모듈 추가는 금지한다.

## 산출물

이번 작업에서 최소한 아래 결과가 있어야 한다.

1. `담당자 기준 조회` 하위 `개인 / 전체` 선택 흐름
2. `전체` 선택 시 담당자 입력 생략
3. `전체` 선택 시 `this_week` 고정
4. `해야 할 일`, `완료` 상태 제외 로직
5. 문서 또는 사용 예시 반영

## 완료 기준

아래 조건을 만족하면 이번 작업을 완료로 본다.

1. `담당자 기준 조회`에서 `개인 / 전체`를 선택할 수 있다.
2. `개인` 선택 시 기존 흐름이 유지된다.
3. `전체` 선택 시 담당자명을 입력받지 않는다.
4. `전체` 선택 시 기간은 자동으로 `this_week`가 적용된다.
5. `전체` 선택 시 기간 선택 화면이 나오지 않는다.
6. `전체` 조회 결과에서 `해야 할 일`, `To Do`, `완료`, `Done` 상태는 제외된다.
7. 기존 프로젝트 조회, 담당자+프로젝트 조회 흐름은 깨지지 않는다.

## 검증 기준

최소한 아래 항목을 검증해야 한다.

1. `담당자 기준 -> 개인` 흐름 확인
2. `담당자 기준 -> 전체` 흐름 확인
3. `전체`에서 담당자 입력을 묻지 않는지 확인
4. `전체`에서 기간 선택을 묻지 않는지 확인
5. `전체` 조회 request/query에 `this_week`가 적용되는지 확인
6. 상태 제외 조건이 실제 JQL 또는 후처리에 반영되는지 확인
7. 기존 다른 조회 모드가 유지되는지 확인
