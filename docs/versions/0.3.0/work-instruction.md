# 작업 지시서: v0.3.0 추가 개발 항목 점검 및 구조 정리

## 문서 목적

이 문서는 `qwen-jira-mcp@0.3.0` 범위에서 검토하고 구현할 추가 개발 항목의 기준 문서다.

현재 배포 기준 버전은 `0.2.1`이며, 이번 문서는 그 다음 작업 단위를 `0.3.0`으로 고정하기 위해 작성한다.

## 버전 문서 구조 원칙

`docs/` 하위 문서는 버전별 디렉토리로 관리한다.

현재 기준 목표 구조:

```text
docs/
  versions/
    0.2.1/
    0.3.0/
      work-instruction.md
```

## 이번 단계 범위

이번 `0.3.0` 범위에서는 아래 항목을 점검하고, 의미 정의가 필요한 부분은 먼저 기준을 확정한다.

1. Jira 이슈 댓글 중 `[주간 이슈]` 태그 댓글 조회 기준 확정
2. 담당자 `전체` 조회 요구의 의미 정의와 구현 가능 범위 점검
3. 프로젝트 조회 시 진척률 계산 기준 논의
4. CLI 첫 진입 흐름을 `Jira 조회`, `댓글 입력`으로 나누는 방향 점검
5. 서버 실행 후 별도 터미널이 필요한 현재 UX 개선 방향 점검

## 항목별 확정 기준

### 1. `[주간 이슈]` 댓글 조회

세부 구현 지시서는 `docs/versions/0.3.0/work-instruction-01-weekly-issue-comment.md`를 따른다.

이 항목은 기존 `[주의사항]` 추출 규칙을 유지하지 않고, 주간 보고용 댓글 규칙으로 전환한다.

확정 기준은 아래와 같다.

1. 공식 태그는 `[주간 이슈]`로 고정한다.
2. 추출 대상은 Jira 이슈 `댓글`만 본다.
3. 이슈 `본문`은 추출 대상에서 제외한다.
4. 댓글 본문에 `[주간 이슈]`가 포함되면 해당 댓글을 주간 이슈 댓글로 간주한다.
5. `[주간 이슈]` 이후 내용을 추출해 사용자에게 보여줄 `주간 이슈` 항목으로 사용한다.
6. 같은 이슈에 `[주간 이슈]` 댓글이 여러 개 있으면 모두 수집한다.
7. 기존 `[주의사항]` 태그와 출력 명칭은 더 이상 사용하지 않는다.

예시:

```text
[주간 이슈]
작업하는데 이런 문제가 있습니다
```

위 댓글은 아래 의미로 처리한다.

```text
주간 이슈:
- 작업하는데 이런 문제가 있습니다
```

## 제외 범위

이번 문서는 아래 항목을 아직 확정하지 않는다.

1. `0.2.1` 문서의 실제 이동 작업
2. `0.3.0` 기능 구현
3. 릴리스 노트 작성
4. 버전 태그 생성과 npm publish

아래 조건을 만족하면 이번 단계 점검 문서는 통과로 본다.

2. 추가 개발 항목 5개가 모두 문서에 반영된다.
