# 후속 작업

## 문서 목적

이 문서는 `qwen-jira-mcp` 배포 이후 이어서 진행할 후속 작업을 정리한다.

현재 상태:

- GitHub 반영 완료
- npm 패키지 `qwen-jira-mcp@0.1.0` 배포 완료
- CLI, local server, MCP 실행 진입점 구성 완료

## 우선순위

### 1. README 최종 정리

목표:

- 실제 배포 패키지 기준 설치/실행 문서로 정리

작업 항목:

1. `npm install -g qwen-jira-mcp` 기준으로 설치 문구 정리
2. `qwen-jira-server`, `qwen-jira`, `qwen-jira-mcp` 실행 예시 추가
3. `.env`와 환경변수 설정 예시 보강
4. `server 먼저 실행 -> CLI 실행` 흐름을 명확히 적기

기대 결과:

- 처음 보는 사용자도 설치 후 바로 실행 가능

### 2. 설치 후 실사용 검증

목표:

- 배포된 npm 패키지가 다른 환경에서도 실제로 동작하는지 확인

작업 항목:

1. 글로벌 설치 검증
2. `qwen-jira-server` 실행 검증
3. `qwen-jira` 실행 검증
4. `.env` 없이 환경변수만으로 동작하는지 확인
5. 오류 메시지가 사용자가 이해하기 쉬운지 확인

기대 결과:

- 배포 패키지의 첫 사용자 경험 검증 완료

### 3. MCP 실제 조회 기능 연결

목표:

- 현재 MCP 진입점을 실제 조회 가능한 흐름으로 확장

작업 항목:

1. MCP가 현재 server API 또는 같은 코어를 사용하도록 정리
2. Jira 조회 tool 입력 스키마 설계
3. 조회 결과 반환 형식 정리
4. 오류 시 MCP 응답 메시지 정리

기대 결과:

- Codex/Claude 같은 에이전트가 실제 Jira 조회 가능

### 4. API/CLI 안정화

목표:

- 사용자 입력, 네트워크 오류, 인증 오류 대응 강화

작업 항목:

1. Jira API 실패 메시지 정리
2. assignee lookup 실패 UX 개선
3. 기간 입력 옵션 정교화
4. 잘못된 프로젝트 키/담당자 입력 시 안내 보강
5. 응답 타임아웃과 재시도 기준 검토

기대 결과:

- 실사용 중 오류 대응 품질 향상

### 5. Markdown 출력 품질 개선

목표:

- 저장되는 Markdown 결과를 더 읽기 좋게 정리

작업 항목:

1. 헤더 구조 정리
2. 이슈별 구분 가독성 개선
3. `[주의사항]` 섹션 강조
4. 파일명 규칙 개선 여부 검토

기대 결과:

- 결과 문서를 바로 공유하거나 보관하기 쉬워짐
