# NestJS Rule

## Encoding

- 설정 파일, 코드, 문서는 모두 `UTF-8` 기준으로 읽는다.
- 설정 파일, 코드, 문서는 모두 `UTF-8` 기준으로 저장한다.

## 목적

NestJS 구현 시 모듈 경계와 책임 분리를 우선한다.

## 구조 원칙

1. 비즈니스 로직은 서비스에 둔다.
2. 외부 시스템 접근은 모듈과 서비스 경계 안에서 분리한다.
3. 설정값과 환경 의존성은 하드코딩하지 않는다.
4. 공통 로직은 재사용 가능한 코어로 모은다.

## 현재 프로젝트 적용 기준

초기 기준 모듈은 아래를 우선한다.

1. `JiraModule`
2. `QueryModule`
3. `SummaryModule`
4. `CliModule`
5. `McpModule`

## 구현 원칙

1. 작업지시서 범위를 넘는 모듈을 임의로 추가하지 않는다.
2. CLI, MCP, 추후 Web은 같은 코어를 재사용하도록 설계한다.
3. Jira 조회는 `Jira API`로 처리한다.
4. `[주의사항]` 같은 규칙 기반 추출은 코드로 처리한다.
5. 명시 지시가 없으면 구조 개선 목적의 대규모 리팩토링을 하지 않는다.
